## 痛点

1. 大纲解析 Agent 错误输出大纲的树形层级或输出顺序相反。例如：先输出子节点，后输出父节点，反复打磨提示词和 check script 无用。

## 暂定解决方案
- A. LLM 调用工具时，让它做 “下一步” 的规划，解决层级问题
- B. 工具锁定输出
- C. 固定输出3层结构，这在大多场景是够用的

2. Node 层级拆分相关问题
问题描述
Node 层级拆分过于细，导致拆出来的 Node 信息密度过低。 若限制 Level，就限制了 AI 的 “灵活”，违背了一开始的设计初衷；不做限制又可能导致信息密度过低，产生大量无用节点，导致 Agent “迷失” 在 outline_tree 中，增加 Token 成本。

## 暂定解决方案

- A. 暂时去掉层级化，直接解析知识点，LLM 输出稳定，基本不会丢失信息，但失去层级后，不好做进度管理，扩展性、灵活性差
- B. 手动构建大纲（针对单一类别场景），适配特定场景（如四六级、升学考试等单一产品），可控性强，但通用性弱，无法适配多场景；整体项目倾向于做通用性质的产品，此方案不符合方向，不过可作为特定场景的解决方案。
